Использование kinect на роботе:

1. Была установлена ROS на Raspberry Pi B+
2. Установлены модули на ROS для работы с Kinect
3. Отдельно установлена библиотека libfreenect
4. Пробовал запускать rviz по сети - не находятся ip адреса, ошибка
5. Библиотеки для работы с libfreenect из python не работают
6. gl-view из примеров libfreenect не работает
7. чтобы заработал freenect-tiltdemo надо сначало запустить файл freenect-micview
8. большинство примеров из libfreenect работает само по себе - не выкидывают ошибки т.е.
9. Примеры, которые окна создают - не хватает мощности RPI, чтобы их отобразить
10. Если запускать команду rostopic echo /camera/rgb/image_raw после запуска freenect, тогда данные отображаются с кинекта
11. https://github.com/OpenKinect/libfreenect/blob/master/examples/regtest.c - этот исходник получает нужные данные и делает из них файлики
12. Запуск команды:
rosrun image_view image_view image:=/camera/rgb/image_color
приводит к ошибке. Описание ошибки здесь: 
https://github.com/ros-perception/image_pipeline/issues/201
13. regtest начинает работать только после wavrecord.  - насколько я понимаю
это загружает какую-то прошивку в кинект и делает его рабочим. Прошивка находится
в audio.bins. audio.bins скачивается из интернета скриптом fwfetcher.py.
14. Модуль kinect_aux просто не работает
15. почти все команды из кинекта не работают, пока не загрзушь прошивку, см здесь:
https://github.com/OpenKinect/libfreenect/issues/376
16. из питона так и не заработало управление мотором, даже после загрузки прошивки
